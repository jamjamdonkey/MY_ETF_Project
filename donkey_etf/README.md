# 🐴 DONKEY ETF — 나만의 29종목 ETF, 정직하게 백테스트하기

> 인덱스를 추종하는 대신, **내가 직접 고른 29종목으로 나만의 ETF를 만들었다.**
> 그리고 그게 진짜 괜찮은지 — 룩어헤드·생존편향까지 까놓고 — 검증하는 백테스트 엔진.

이 레포는 내 블로그 *DONKEY-LAB*의 「DONKEY ETF 백서(1·2·3편)」를 떠받치는 코드다. 백서에 나오는 모든 차트가 여기서 나온다.

---

## 🎯 이게 뭐냐면

나는 매일 일정 금액을 4개의 증권사 계좌(슬리브)에 나눠 29개 종목에 적립식으로 넣는다. 이걸 하나의 **"수제 ETF"** 라고 부른다. 종목은 그냥 꽂은 게 아니라 **오디션 규칙**으로 골랐다 — 해자가 무너지고 자본배분이 4분기 이상 망가지면, 손익과 상관없이 잘라낸다.

근데 "내가 고른 게 좋다"는 건 누구나 한다. 중요한 건 **그게 진짜인지 데이터로 까보는 것**이고, 더 중요한 건 **그 검증이 정직한지**다. 그래서 이 엔진은 화려한 수익률 자랑이 아니라, 벤치마크(QQQ/SPY/SCHD) 대비 **위험조정성과·낙폭·상관구조·위기 대응**을 3단계로 해부한다.

## 📂 구조

```
donkey_etf/
├── config.py          # 전역 설정 (폰트/벤치마크/기간/무위험수익률/출력경로)
├── portfolio.py       # ★ 단일 진실 공급원: 29종목 · 4슬리브 · 일일 금액
├── data.py            # 데이터 계층: 다운로드 + Adj Close 추출 + Cash Drag (+오프라인 폴백)
├── metrics.py         # 퀀트 지표: CAGR/MDD/Sharpe/Sortino/캡처/VaR/리스크기여도 …
├── analysis/
│   ├── a1_individual.py   # [레벨1] 개별 29종목 — 배수 추이 + 상관 히트맵
│   ├── a2_sleeves.py      # [레벨2] 4개 ETF(슬리브) — 누적수익 + 효율적투자선 + PCA
│   └── a3_portfolio.py    # [레벨3] 전체 29종목 — 티어시트 + 스트레스 + 롤링 + 캡처
├── run_all.py         # 전체 실행 (레벨1 → 2 → 3)
└── output/            # 생성된 차트(PNG)
```

핵심은 **`portfolio.py` 하나**다. 비중을 여기서 '일일 적립 금액(원)' 으로만 정의하고, 종목 비중·슬리브 비중·티커 목록을 전부 여기서 파생한다. **한 줄만 고치면 전체 분석에 반영**된다.

## 🧠 방법론 — '오디션'으로 종목을 고른다

- 매수는 **적립식 분산**. 한 번에 몰빵하지 않는다.
- 보유는 **조건부**. 해자(moat)가 무너지고 + 자본배분/주주환원이 4분기 이상 어긋나면 → **손익과 무관하게 청산**하고 적립도 멈춘다.
- 미래엔 추가 자금을 **인덱스**로 — 코어(인덱스) + 위성(이 액티브 슬리브) 구조로 간다.

즉 이건 '천재적 종목 발굴'이 아니라 **규칙 기반의 자기 점검 시스템**이다.

---

## ⚠️ 정직하게 — 이 백테스트의 한계

내 모든 분석 글의 1번 원칙이다. 좋아 보이는 숫자보다 **편향을 먼저 깐다.**

- **룩어헤드 편향(Look-ahead bias)** — '지금 좋아 보이는 29종목'을 과거에 들고 있었다고 가정하는 순간, 이미 미래를 알고 고른 셈이다. 이 백테스트의 수익률은 그래서 **과대평가될 수밖에 없다.**
- **생존 편향(Survivorship bias)** — 살아남아 지금 내 포트에 있는 종목만 본다. 중간에 잘려나갔을 종목은 빠져 있다.
- **과거 ≠ 미래** — 과거 성과가 미래 수익을 보장하지 않는다. 이건 *검증 도구*이지 *예측 도구*가 아니다.
- 본 코드는 **개인 투자 기록·연구용**이며, 특정 종목 추천이나 투자 권유가 아니다.

> **오프라인 폴백 안내** — 야후 접속이 막힌 환경(회사망 등)에서는 `data.py`가 **합성(synthetic) 데이터**로 폴백해 파이프라인 구조만 검증한다(콘솔에 경고가 뜨고, 이때 숫자는 무의미). **실제 백테스트 수치는 본인 PC에서 `python run_all.py`** 를 돌리면 yfinance 실데이터로 나온다.

## 🔧 리팩토링 기록 — 흩어진 18개 스크립트를 하나로

원래 이 분석은 복붙으로 늘어난 **18개 .py 파일**이었다. 정리하면서 실제 버그도 잡았다:

1. **비중 불일치 통일** — 같은 포트폴리오인데 파일마다 비중을 3가지로 적어 서로 안 맞았다(정수 capital 16/14/5/8, 금액 4000/43000). → `portfolio.py`에서 **일일 금액(원)을 유일 기준**으로 삼아 전부 파생.
2. **중복 코드 제거** — 다운로드/Adj Close 추출/MDD/폰트 설정이 18곳에 복붙돼 있던 걸 모듈 4개로 통합.
3. **Cash Drag 일관화** — 상장 전 구간을 '0'이 아니라 **무위험수익률**로 채워(현금 보유 가정) 가중합 왜곡 제거.
4. **오프라인 폴백** — 야후가 막혀도 합성 데이터로 구조 검증이 가능하게.

> 확장도 같은 패턴이다. 6팩터·매크로 민감도·FX 효과·K-means 복제 같은 분석도 코어(`portfolio/data/metrics`)를 그대로 재사용해서 `analysis/`에 모듈만 추가하면 된다.
